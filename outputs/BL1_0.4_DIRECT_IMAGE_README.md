# K3765-Z BL1 0.4 direct Linux Image handoff

BL1 0.4 bypasses the zImage decompressor that currently exceeds the MSM6290
watchdog window. It loads the verified uncompressed ARM `Image` directly at
its final physical entry address and branches into Linux `head.S`.

Everything remains RAM-only. Neither BL1 nor the host loader implements a
NAND erase, partition, program, or flash-write operation.

## Verified inputs

| Item | Address | Size | Verification |
|---|---:|---:|---|
| Stage-0 monitor | `0x00800000` | 105,928 | stock-sized image, PBL ACKs, expected monitor banner |
| Linux `Image` | `0x00208000` | 3,569,520 | SHA-256 `6a67cad42756124f7aa084e1d625c779bb39e1408e3f69648784010b35b10f51` |
| BL1 0.4 | `0x01000000` | 5,933 | CRC32 `F0E0ADF7` |
| DTB | `0x01F80000` | 549 | CRC32 `5D395650` |

The Linux `Image` ends at `0x0056F770`, below the resident monitor at
`0x00800000`. BL1 checks the exact Image size and 17 sparse 32-bit
fingerprints, including the first four entry instructions and final word,
before the non-returning branch.

## Load from a fresh downloader

```powershell
py -3.9 .\k3765_ram_bundle_load.py COMxx `
  .\armprg_stage0_monitor.bin `
  --image "0x00208000:.\Image-k3765-probe" `
  --image "0x01000000:.\k3765_bl1_linux_image.bin" `
  --image "0x01F80000:.\k3765-z-probe.dtb" `
  --log .\direct_image_bundle_load.log
```

Verify the monitor, BL1, and DTB before arming the handoff:

```powershell
py -3.9 .\k3765_stage0_console.py COMyy info
py -3.9 .\k3765_stage0_console.py COMyy crc 0x01000000 5933
py -3.9 .\k3765_stage0_console.py COMyy crc 0x01F80000 549
```

Only after those checks match, invoke BL1 with the explicit `IMG1` marker:

```powershell
py -3.9 .\k3765_stage0_console.py COMyy `
  linux 0x01000000 3569520 549 0x494D4731 `
  --idle-timeout 30 `
  --log .\bl1_0.4_direct_image_boot.log
```

Expected first kernel milestone:

```text
JUMPING DIRECTLY TO DECOMPRESSED LINUX IMAGE
Shadow-MSM: entered decompressed Linux head.S
```

The first target run also reached processor recognition and initial page-table
creation. Its next trace initially faulted because Linux had already
repurposed `sp` as the future virtual `__mmap_switched` address. The revised
kernel probe uses a dedicated physical trace stack in the monitor's reserved
RAM window and adds individual ARM926 setup milestones.

The handoff is non-returning. A watchdog reset or power cycle restores the
stock firmware because no persistent storage is modified.
