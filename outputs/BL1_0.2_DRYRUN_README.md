# K3765-Z BL1 0.2 Linux handoff dry run

**Verified on physical K3765-Z hardware on 2026-07-20.**

This bundle is RAM-only. The loader implements legacy PBL RAM write and execute
commands only; it contains no NAND erase, program, partition, or flash-write
operation.

## Built artifacts

- `armprg_stage0_monitor.bin` at `0x00800000`
- `k3765_bl1_linux_dryrun.bin` at `0x01000000`
- `k3765_zimage_header_fixture.bin` at `0x01200000`
- `k3765_minimal.dtb` at `0x01F80000`

The zImage fixture is only a 64-byte header parser test. It is deliberately not
executable. BL1 0.2 validates it and the DTB, prints the future Linux handoff,
and returns to stage-0 without jumping.

## Expected checks

| Artifact | Size | CRC32 |
|---|---:|---:|
| BL1 0.2 before execution | 5,677 | `83FD9148` |
| BL1 0.2 after successful dry run | 5,677 | `1AFE6067` |
| zImage header fixture | 64 | `099A07E1` |
| minimal DTB | 429 | `B82041C8` |

The BL1 post-run CRC differs because its 13-byte hexadecimal formatting buffer
ends with `0x00051078`, the last value printed. Executable code is unchanged.

## Load from a fresh PBL downloader session

Run from the project directory, replacing `COM44` if Windows assigns a new
downloader port:

```powershell
py -3.9 .\outputs\k3765_ram_bundle_load.py COM44 `
  .\outputs\armprg_stage0_monitor.bin `
  --image "0x01000000:.\outputs\k3765_bl1_linux_dryrun.bin" `
  --image "0x01200000:.\outputs\k3765_zimage_header_fixture.bin" `
  --image "0x01F80000:.\outputs\k3765_minimal.dtb" `
  --log .\outputs\bl1_0.2_bundle_load.log
```

## Verify target RAM before execution

```powershell
py -3.9 .\outputs\k3765_stage0_console.py COM44 crc 0x01000000 5677
py -3.9 .\outputs\k3765_stage0_console.py COM44 crc 0x01200000 64
py -3.9 .\outputs\k3765_stage0_console.py COM44 crc 0x01F80000 429
```

Expected:

```text
CRC32: 0x83FD9148
CRC32: 0x099A07E1
CRC32: 0xB82041C8
```

Do not call BL1 if any CRC differs.

## Run the non-jumping handoff dry run

The three arguments are zImage size, DTB size, and a visible dry-run marker:

```powershell
py -3.9 .\outputs\k3765_stage0_console.py COM44 `
  boot 0x01000000 64 429 0x44525931 `
  --log .\outputs\bl1_0.2_dryrun.log
```

The expected final line is:

```text
RETURN R0: 0x44525931
```

This return value is ASCII `DRY1`. A return proves that no branch to the fixture
occurred.

## Verified target result

```text
zImage magic   : 0x016F2818
zImage hdr start: 0x00108000
zImage hdr end : 0x00108040
DTB magic      : 0xEDFE0DD0
DTB total size : 0x000001AD
PC              : 0x01200000
r0              : 0x00000000
r1              : 0xFFFFFFFF
r2              : 0x01F80000
Validation PASS
DRY RUN COMPLETE; no branch to Linux was performed
RETURN R0: 0x44525931
```

The post-run BL1 CRC was exactly `0x1AFE6067`, and a complete stage-0 system
query succeeded afterward.
