# K3765-Z BL1 0.3 Linux boot handoff

**Status: host-built and statically audited; not yet executed on the target.**

BL1 0.3 is the first non-returning Linux handoff stage. It is still entirely
RAM-resident and contains no NAND, partition, erase, or flash-program command.

## BL1 artifact

| Property | Value |
|---|---|
| File | `k3765_bl1_linux_boot.bin` |
| Load/entry | `0x01000000` |
| Size | 5,933 bytes |
| SHA-256 | `2daa4165adf338733e0d15f00f0bf254244feb4f79c6125aa5f91f094c942c2f` |
| CRC32 | `F11D72BA` |
| Arming marker | `0x4C4E5831` (`LNX1`) |

BL1 validates:

- the explicit `LNX1` arming marker;
- zImage and DTB sizes against their reserved windows;
- ARM zImage magic `0x016F2818`;
- FDT magic and exact total size;
- MMU disabled;
- data cache disabled; and
- the planned Linux entry registers.

On success it selects the blue LED checkpoint, masks IRQ/FIQ, drains the write
buffer, invalidates the instruction cache and TLB, and enters:

```text
PC = 0x01200000
r0 = 0x00000000
r1 = 0xFFFFFFFF
r2 = 0x01F80000
```

## Mandatory preflight

Do not execute this BL1 with the 64-byte header fixture. Wait for a verified
`zImage-k3765-probe` and `k3765-z-probe.dtb` from the kernel build.

Before the handoff, all four target-side CRCs must match their host values:

```text
stage-0 monitor
BL1 0.3
real zImage
real DTB
```

Any mismatch cancels the test.

## Non-returning log follower

The host console now has a `linux` action. It forwards every diagnostic frame
until the transport disconnects, BL1 unexpectedly returns, or the idle timeout
expires:

```powershell
py -3.9 .\outputs\k3765_stage0_console.py COMxx `
  linux 0x01000000 ZIMAGE_SIZE DTB_SIZE 0x4C4E5831 `
  --idle-timeout 30 `
  --log .\outputs\bl1_0.3_linux_boot.log
```

Expected pre-MMU sequence:

```text
K3765-Z BL1 0.3 Linux boot
Validation PASS
LED checkpoint  : blue
JUMPING TO LINUX; BL1 WILL NOT RETURN
Uncompressing Linux...
 done, booting the kernel.
Shadow-MSM: entered decompressed Linux head.S
Shadow-MSM: ARM926 processor lookup passed
Shadow-MSM: initial page tables created
Shadow-MSM: enabling the Linux MMU next
```

The stock runtime is restored by physically power-cycling the modem.
